## **Overview 🎯**

[Streamlit WhatsApp Messaging Application](https://chatgpt.com/c/67ca7156-0568-800e-bd1c-fa346667ced8)
[Complete Script](https://chatgpt.com/c/67ca7156-0568-800e-bd1c-fa346667ced8)

The Streamlit WhatsApp Messaging Application is a web-based tool designed to send automated WhatsApp messages to individual contacts and groups. It supports bulk messaging, scheduled messages, and multimedia sharing, making it ideal for marketing campaigns, notifications, and personal communication. The application is built using Python, Streamlit, and PyWhatKit, ensuring seamless WhatsApp Web automation.

## **Objectives🎯**

1- Develop an easy-to-use web interface using Streamlit.

2- Automate WhatsApp messaging with PyWhatKit.

3- Enable bulk messaging to contacts and groups.

4- Support scheduled messages for future delivery.

5- Allow sending of multimedia (images, videos, documents).

6- Provide message logs and status tracking.

7-Order & Payment Notifications 💰

## **System Architecture🎯**

The system consists of three main components:

1. **Frontend (Streamlit Web App)**

 - User-friendly dashboard for managing messages.

 - Input fields for recipient numbers, message text, and attachments.

 - Schedule messages using a simple UI..

2. **Backend (Python & PyWhatKit)**

- Uses PyWhatKit to send messages via WhatsApp Web.

- Handles scheduling of messages.

- Supports logging and error handling..

3. **Database (Optional - MySQL / Firebase)**

- Stores sent messages, delivery status, and logs.

- Allows retrieving message history for analysis.

## **Key Features 📌**

✅ Send messages to individual WhatsApp contacts.

✅ Bulk messaging to multiple contacts.

✅ WhatsApp group messaging for streamlined communication.

✅ Schedule messages for future delivery.

✅ Supports multimedia (images, videos, documents).

✅ User-friendly web interface with Streamlit.

✅ Message logs and tracking for sent messages.

✅ Order & Payment Notifications 💰-Send auto-updates on order status or payment reminders.

✅ WhatsApp Chatbot for FAQs 🤖-Reply automatically to customer inquiries (e.g., business hours, pricing).

✅ Upload CSV File: Users can upload a CSV containing phone numbers.

✅ Automatic Parsing: The app extracts valid phone numbers from the file.

✅ Supports Manual Input: Users can still enter numbers manually if they prefer.

## **Technology Stack📂**

1. **Frontend** : Streamlit (Python Web Framework).
2. **Backend**: Python + PyWhatKit (for WhatsApp messaging),pyautogui(pyautogui is a Python library used for automating mouse and keyboard actions.),Twilio API .,Selenium
3. **Database (Optional)**: MySQL / Firebase (for message storage).
4. **Hosting** - Can be deployed on Streamlit Cloud, Heroku, or a local server

## 💻 Workflow for Contributions:

1-**Students Fork the Repository**

- Each student forks your GitHub repo to their account.
- They work on their portion in their own forked repo.
  
2- **Students Create Feature Branches**
  
- Instead of working directly on main, they create a feature branch
  
3- **SCommit & Push Changes**S
  
- Students make changes, commit them, and push to their branch

4- **Submit Pull Requests (PRs)**

- Once they complete their portion, they submit a Pull Request (PR) to your main repository.
- As the project owner, you review their PRs and merge them after approval.

## ⚙️ **Tracking Student Contributions**

- Use GitHub Issues: Assign tasks to students using GitHub issues.
- Use Project Boards: GitHub Projects can help track progress.
- Use Pull Request Reviews: To ensure code quality before merging.
- Set Milestones: Define deadlines for each task.
  
## ⚙ **Finalizing the Project**
Once all contributions are merged:
- Conduct a final review and clean up unnecessary files.
  
Generate a final report including
- Contributions by each student.
- A working demo.
- Documentation for future users.
